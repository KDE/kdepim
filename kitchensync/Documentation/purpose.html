Purpose

The purpose of KitchenSync is to provide the user with a way of choosing which device should be Synced -- since devices may share a communication connection with the machine -- and displaying information about the Sync. To make this possible, KitchenSync has an extensive DCOP interface that makes it possible for daemons to:

  • Request a specific communication port (this should be denied if another daemon is busy with that port).

  • Indicate what kinds of Sync possibilities (sync, backup, restore, etc.) the daemon has.

  • Start a Sync and display the progress of the Sync.

  • Log information about a Sync.
