KitchenSync startup

  1. KitchenSync starts and adds an icon to the system tray with some kind of generic Sync icon. The KPilot Sync icon doesn't seem suitable since it's the Grüne Punkt of German recycling, and this may be confusing. Something that resembles the Palm HotSync icon may be better.

  2. KitchenSync queries for applications with ServiceType SyncDaemon. Their names are added to a popup menu.

  3. When the user selects a daemon, it is run.

  4. When a daemon is run, it registers with KitchenSync. This might be refused if there is already a daemon running or somesuch -- obviously you can run a daemon from the command line as well.

  5. The daemon tells KitchenSync what kind of Syncs it can do. These are added to the popup menu. A daemon might respond to data coming from the device to start a Sync, or it may start a Sync programmatically. This might be configured as well.

  6. KitchenSync might show a progress bar window, or a window with a Sync log of messages.

  7. If the user selects a different daemon, the current daemon is told to quit and the new one is started from step 3.
